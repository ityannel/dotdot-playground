# 🎤 渋谷＆シリコンバレー合同調査！「100人に聞いた PopPop vs 既存言語」街角アンケート結果報告書

**「忖度なしの生の声を聞かせてください！」**

今回は、ITエンジニアから非エンジニアまで、計100人の方に **PopPopの構文** と **既存言語（Python/JavaScript/Rustなど）の構文** を比較していただき、どちらが好きか、どう感じたかをガチでアンケートしてきました！

## 📊 対象者（計100名）
1. **プログラミング初心者・非コーダー（学生、マーケター、ビジネス層）：30名**
2. **Web/フロントエンド/JSエンジニア：30名**
3. **Python/データサイエンティスト：20名**
4. **C++/Rust/低レイヤー過激派（ピュアリスト）：10名**
5. **UI/UXデザイナー・PdM：10名**

---

## 💻 比較に使用したコード

**PopPopの構文:**
```poppop
(編集データ) >> (@, 編集データ1) >> Solve >> "結果が出ました: {@}" >> Display >!> "うまく表示できませんでした" >> Display.
```

**既存言語のイメージ（例: JS/Python等）:**
```javascript
try {
  const result = solve(editData, editData1);
  display(`結果が出ました: ${result}`);
} catch (e) {
  display("うまく表示できませんでした");
}
```

## 🗣️ 生の声・ガチレビューまとめ（属性別）

### 1. プログラミング初心者・非コーダー（30名）
**【支持率】 PopPop派 25人 / 既存言語派 5人**

> 「矢印 (`>>`) でデータが右に流れていくのがめちゃくちゃ分かりやすい！普通のコードはカッコや英単語が多くてウッとなるけど、これなら読める」（20代・文系学生）

> 「`@` が何なのか一瞬戸惑ったけど、『直前の結果』って言われたら納得。パズルみたいで楽しそう」（30代・マーケター）

> 「エラー処理が `>!>` なのは顔文字みたいで可愛い。既存の `try-catch` って何回聞いても覚えられなかったので、こっちが良い！」（20代・広報）

**総評:** 圧倒的にPopPopが支持されました。直感的なデータの流れ（パイプライン）が、プログラミング特有の「ネスト（入れ子）の壁」をぶち壊しているようです。

### 2. Web/フロントエンド/JSエンジニア（30名）
**【支持率】 PopPop派 18人 / 既存言語派 12人**

> 「JSのPromiseチェーン (`.then()`) の究極進化系って感じ。コールバック地獄から完全に解放されそうで良いね！」（30代・フロントエンド）

> 「エラーハンドリングの `>!>` は面白い。ただ、キャッチしたエラーの型や詳細メッセージ（スタックトレースなど）はどうやって取り出すの？それが出来ないと実務じゃキツいかな」（20代・Reactエンジニア）

> 「結局はシンタックスシュガーだから、デバッグツールが優秀じゃないと使う気にならない。でも書いてて気持ちよさそう」（40代・テックリード）

**総評:** 概ね好印象ですが、実務視点からの厳しいツッコミ（エラー詳細の取得、デバッガの有無など）も目立ちました。

### 3. Python/データサイエンティスト（20名）
**【支持率】 PopPop派 12人 / 既存言語派 8人**

> 「データ処理のパイプラインを書くのにすごく向いてそう。Pandasのメソッドチェーンに近い感覚で書けるのは最高」（30代・データエンジニア）

> 「暗黙の変数 `@` は、短いコードならいいけど、長い処理になると『これ今何が入ってんの？』って絶対なる。可読性を下げる原因にならないか心配」（40代・機械学習エンジニア）

**総評:** データフロー指向との相性の良さが評価される一方で、「暗黙の状態」を嫌う意見も。

### 4. C++/Rust/低レイヤー過激派（10名）
**【支持率】 PopPop派 2人 / 既存言語派 8人**

> 「は？ マジック変数 `@` なんて許せない。スコープはどうなってるんだ？ メモリ管理はどうやってるんだ？ 完全にブラックボックスじゃないか！」（30代・C++er）

> 「暗黙的な挙動が多すぎる。コードは明示的であるべきだ。Rustなら `Result<T, E>` でコンパイラが全て教えてくれる。`>!>` なんて甘えだ」（20代・Rustacean）

> 「（少数派の賛成意見）裏でちゃんとモナドっぽくエラーハンドリング（Result型）されてるなら、シェルスクリプトの代替としてアリかも？」（40代・組み込みエンジニア）

**総評:** 案の定、過激派からは大ブーイングの嵐でした。彼らには「隠蔽された便利さ」よりも「完全なコントロール」が必要です。

### 5. UI/UXデザイナー・PdM（10名）
**【支持率】 PopPop派 8人 / 既存言語派 2人**

> 「コードというより、Figmaやノードベースのツールのワイヤーフレームを見てるみたい。エンジニアが何をやろうとしてるか、非エンジニアでも追えるのはチーム開発で大きな武器になる！」（30代・PdM）

> 「視覚的に流れが見えて美しい。記号の選び方（`>>`, `>!>`）にUI的なセンスを感じる」（20代・UIデザイナー）

**総評:** 非コード領域の職種からは「コミュニケーションツール」としての価値が高く評価されました。

## 🏆 最終結果発表：PopPop vs 既存言語

**アンケート集計結果：**
- 🌟 **PopPop派：65% （65人）**
- 🏛️ **既存言語派：35% （35人）**

### 🎉 結論
**「PopPopは、初心者を味方につけ、過激派を敵に回す、愛され・尖り言語である！」**

可読性と直感性において、PopPopは従来のパラダイムを圧倒するポテンシャルを見せつけました。一方で、低レイヤー志向のエンジニアからは「暗黙的すぎる」と敬遠される結果に。
今後は「いかにデバッグしやすくするか」「暗黙の変数 `@` をどう安全に扱うか」が、エンタープライズ普及への鍵となりそうです！
