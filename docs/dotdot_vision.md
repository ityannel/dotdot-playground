# PopPop Language: Comprehensive Vision & Roadmap

PopPopの「Left-to-Right Dataflow」「Tacit Programming」という中核哲学を極限まで押し広げた場合、言語としてどのような進化が可能か。網羅的に思考を展開した未来の機能・構想リストです。

## 1. パイプラインによる分配代入 (Destructuring Assignment)
現在、配列の要素を別々の変数に保存するにはインデックスアクセスが必要ですが、バンドル（`&`）を「受け皿」として使うことで直感的な分配代入が可能になります。

```poppop
// 配列の要素を左から右へ、それぞれの変数へ一気に流し込む
[100, 200] >> (x & y).

// 関数から複数の戻り値（バンドル）を返し、それぞれを受け取る
("Alice,25" & ",") >> Split >> (name & age).
```
**哲学**: 代入（`=`）を使わず、データの流れの終着点として複数の変数（バンドル）を配置する美しい設計。

## 2. カリー化と部分適用 (Currying & Partial Application)
パイプライン処理と非常に関性が高いのが「カリー化」です。関数が要求する引数（バンドル）の数が足りない場合、エラーにするのではなく「残りの引数を待つ新しい関数」を生成してパイプへ流します。

```poppop
(a & b) >> new Add:
    a + b >> Return..
..

// 引数が1つしか流れてこない場合、「あと1つ受け取ったら足し算する関数」になる
10 >> Add >> addTen.

// 後からもう1つの引数を流し込む
5 >> addTen >> Display. // 結果: 15
```
**哲学**: すべての関数がパイプラインの単一の部品（フィルタ）として柔軟に使い回せるようになります。

## 3. ストリームの並べ替えとグループ化 (`Sort:` / `Group:`)
`Map:` や `Filter:` に加え、データサイエンスや実務で頻出する操作もストリームブロックとして提供します。

```poppop
users >> Sort @age:
    // 年齢の昇順でソート
    @age >> Return..
.. >> Group @category:
    // カテゴリごとに辞書（Map）化
    @category >> Return..
.. >> Display.
```
**哲学**: SQLのような宣言的データ操作を、パイプラインの流れの中で完全にシームレスに記述。

## 4. 明示的なエラーの送出とパイプの破壊 (`Throw` / `Fail`)
`>~>`（例外キャッチ）と対になる機能として、パイプラインの任意の箇所で「意図的にエラーを発生させ、通常のフローをへし折る」ための組み込み関数です。

```poppop
user >> check:
    is null: "User not found!" >> Throw.
    else:    @::name >> Return..
.. >~> "Guest" >> Display.
```
**哲学**: 例外もまた、パイプラインを逆流するのではなく、専用のエラー用パイプ（`>~>`）へと流れ込む「データ」の一種として扱います。

## 5. 文字列への暗黙変数展開 (String Interpolation)
パイプラインを流れる現在のデータ（`@` や `@name`）を、文字列の中に直接埋め込めるようにします。

```poppop
{"name": "Alice", "age": 20} >> $"Hello, {@::name}! You are {@::age} years old." >> Display.
```
**哲学**: データのフォーマット処理すらも、直感的な一本のパイプラインで完結させます。

---

これらの構想はすべて、PopPopの「視線の動き（左から右）とデータの動きを完全に一致させる」という設計思想に完璧に合致します。
これらが揃えば、汎用言語としての表現力で既存のモダン言語（RustやSwiftなど）に引けを取らない、圧倒的にエレガントな言語が完成します。
